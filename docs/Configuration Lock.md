# Configuration Lock Guide (Linux)

This guide explains how to manually enable and validate Biopass configuration lock.

Biopass does not enable configuration lock automatically.

## What Configuration Lock Does

Configuration lock protects Biopass by requiring a master key that is verified against a stored hash in:

- `/usr/lib/biopass/secret.yaml`

The file is generated by `biopass_key_generator.py` and validated by `biopass_key_test.py`.

## Prerequisites

1. Make sure Biopass is already working for your use case before enabling lock.
2. Install the Python dependency used by both scripts:

```bash
python3 -m pip install argon2-cffi
```

3. Open a terminal at the scripts directory in this location:

```bash
cd /usr/lib/biopass
```

## 1. Generate Lock Secret (Run as root)

Run the generator with root privileges:

```bash
sudo ./biopass_key_generator.py
```

You will be prompted to enter and confirm a master key. On success, the script writes `/usr/lib/biopass/secret.yaml`.

Important:

- Keep the master key private and recoverable.
- If you lose the key, you cannot validate configuration lock without regenerating the secret file.

## 2. Test the Master Key (Run as normal user)

Do not run the tester as root. The script rejects root execution.

Interactive test:

```bash
./biopass_key_test.py
```

Expected success output:

```text
SUCCESS: Master key is valid.
```

Machine-friendly output:

```bash
./biopass_key_test.py --result-only
```

Returns `success` when valid, `error` when invalid.

## 3. Optional Non-Interactive Test Modes

`biopass_key_test.py` also supports:

- `--password "<key>"` (least secure)
- `--password-stdin`
- `--password-env ENV_VAR`

Examples:

```bash
printf '%s' "your-master-key" | ./biopass_key_test.py --password-stdin --result-only
```

```bash
BIOPASS_MASTER_KEY="your-master-key" ./biopass_key_test.py --password-env BIOPASS_MASTER_KEY --result-only
```

## Troubleshooting

- `ERROR: Missing dependency 'argon2-cffi'.`
  Install dependency with `python3 -m pip install argon2-cffi`.
- `ERROR: Secret file not found at /usr/lib/biopass/secret.yaml.`
  Re-run `sudo ./biopass_key_generator.py`.
- `ERROR: BioPass Key Tester must not be run as root.`
  Run tester as your regular user.

## Security Notes

- Prefer interactive entry or `--password-stdin` over `--password` to reduce exposure in process lists.
- Restrict access to terminals, shell history, and environment variables that may contain the master key.
